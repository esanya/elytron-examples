This example demonstrates how WildFlyClientDefaultSSLContextProvider can be used to register JVM wide default SSLContext.

There is no user interface for this quickstart. Instead, you can verify the REST endpoint is up and running by accessing the following URL:

rest/hello

This URL will prompt you to accept server's certificate and provide client's certificate. Run test from the client module associated with this example to see that WildFlyClientDefaultSSLContextProvider was used to provide default SSLContext to the client.
